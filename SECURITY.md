# Политика безопасности

## Поддерживаемые версии

| Версия | Поддерживается |
|--------|----------------|
| 2.x    | Да             |
| 1.x    | Нет            |

## Сообщение об уязвимости

Проект обрабатывает только текст и hex-цвета, переданные пользователем, для рендеринга изображений на серверном canvas. Нет сетевых запросов, нет записи файлов вне папки `output/`, пользовательский ввод не выполняется как код.

Если вы обнаружили уязвимость (например, path traversal при загрузке шрифтов или формировании имён файлов), сообщите об этом через GitHub Issue с меткой **[SECURITY]** или напрямую мейнтейнеру.

Пожалуйста, не открывайте публичный issue для уязвимостей, которые можно эксплуатировать до выхода фикса.

## Область действия

- Загрузка шрифтов — только из подпапки `fonts/`
- Запись файлов — только в подпапку `output/`
- Никаких внешних сетевых запросов во время работы
- Никакого `eval` или динамического выполнения кода

## Зависимости

Следите за актуальностью `canvas` и `typescript`. Регулярно запускайте `npm audit` для проверки известных уязвимостей в зависимостях.
